Step 1 — Preparing Your Floor Plan

Getting a Clean Plan from RoomSketcher

RoomSketcher (and similar tools like Planner 5D, Floorplanner, or even a hand-drawn plan scanned flat) produces the background image layer. The web UI doesn't need a special format — a clean PNG screenshot is all it needs. The sensor positions are stored separately in a JSON config file, so the image never needs to be re-exported when sensors are added or moved.

📐
What makes a good source floor plan Draw one floor at a time in RoomSketcher. Use 2D top-down view — not 3D. Remove furniture if it makes room boundaries harder to see. Export at 1200–2000px wide minimum for crisp rendering on high-DPI screens. Name each exported PNG by floor: floor_basement.png, floor_1.png, floor_2.png.
📏
The most important step: consistent dimensions Before marking any sensor positions, decide on a final crop and resolution for each floor plan image and stick to it. Sensor positions are stored as percentages of image width and height — if you re-export the image at a different crop, all positions shift. Export once, save the original, use that file as-is forever.

Step 3 — Heat Map Rendering

How the Temperature Overlay Works

The heat map is rendered in real time on an HTML5 Canvas element that sits directly on top of your floor plan image. It uses Inverse Distance Weighting (IDW) — a simple algorithm that interpolates the temperature at any pixel based on its distance from all known sensor readings. Sensors closer to a pixel have more influence than sensors further away.

🧮
Why IDW is the right algorithm here It doesn't require training data, it uses every sensor you have regardless of count, new sensors automatically integrate, and you can tune one parameter (the power factor) to control how sharply the gradient transitions between sensor readings. A power of 2 gives smooth, natural-looking gradients. A power of 3–4 makes each sensor more "local," which is physically more accurate for separate rooms. 
// JavaScript — floor plan canvas renderer
// Runs in the browser as part of the FastAPI-served web UI

const COLOR_SCALE = [
  { temp: 55, color: [70, 130, 200] },   // cold blue
  { temp: 65, color: [100, 180, 150] },  // teal — comfortable cool
  { temp: 70, color: [122, 196, 160] },  // green — ideal
  { temp: 74, color: [220, 160,  80] },  // amber — warm
  { temp: 80, color: [220,  90,  50] },  // orange — hot
];

function tempToColor(tempF) {
  for (let i = 0; i < COLOR_SCALE.length - 1; i++) {
    const lo = COLOR_SCALE[i], hi = COLOR_SCALE[i+1];
    if (tempF >= lo.temp && tempF <= hi.temp) {
      const t = (tempF - lo.temp) / (hi.temp - lo.temp);
      return lo.color.map((c, j) => Math.round(c + t * (hi.color[j] - c)));
    }
  }
  return COLOR_SCALE[COLOR_SCALE.length - 1].color;
}

function idwInterpolate(px, py, sensors, power = 2) {
  // px, py: pixel position as fraction of canvas width/height
  let weightedSum = 0, totalWeight = 0;
  for (const s of sensors) {
    const dx = px - s.x / 100;
    const dy = py - s.y / 100;
    const dist = Math.sqrt(dx*dx + dy*dy);
    if (dist < 0.001) return s.temp; // at the sensor itself
    const w = 1 / Math.pow(dist, power);
    weightedSum += w * s.temp;
    totalWeight += w;
  }
  return weightedSum / totalWeight;
}

function renderHeatmap(canvas, sensors, alpha = 0.45) {
  const ctx = canvas.getContext('2d');
  const { width, height } = canvas;
  const step = 6;  // pixel step — lower = smoother but slower
  const imgData = ctx.createImageData(width, height);

  for (let y = 0; y < height; y += step) {
    for (let x = 0; x < width; x += step) {
      const temp  = idwInterpolate(x/width, y/height, sensors);
      const [r,g,b] = tempToColor(temp);
      // Fill a step×step block for performance
      for (let dy = 0; dy < step && y+dy < height; dy++) {
        for (let dx = 0; dx < step && x+dx < width; dx++) {
          const idx = ((y+dy) * width + (x+dx)) * 4;
          imgData.data[idx]   = r;
          imgData.data[idx+1] = g;
          imgData.data[idx+2] = b;
          imgData.data[idx+3] = Math.round(alpha * 255);
        }
      }
    }
  }
  ctx.putImageData(imgData, 0, 0);
}

// Refresh every 30 seconds when sensor data updates via MQTT
async function refreshFloorPlan() {
  const data    = await fetch('/api/indoor').then(r => r.json());
  const config  = await fetch('/api/floor_plan_config').then(r => r.json());
  const floor   = config.floors.find(f => f.id === currentFloor);

  const sensors = floor.sensors.map(s => ({
    x: s.x, y: s.y,
    temp: data[s.id]?.temperature ?? 68
  }));

  renderHeatmap(document.getElementById('heatmap-canvas'), sensors);
  updateMarkers(floor, data);
}
Step 4 — Placing Sensors on the Map

Built-in Position Editor

Rather than manually measuring pixel coordinates, the dashboard includes a one-time setup tool that lets you click directly on the floor plan image to place each sensor. It writes the percentages back to floor_plan_config.json automatically.

# Access the position editor at:
http://your-server:8080/setup/floor-plan

# Editor workflow:
# 1. Select a floor from the dropdown
# 2. Select a sensor or equipment item from the sidebar list
# 3. Click its location on the floor plan image
# 4. x/y percentages are calculated and saved automatically
# 5. Repeat for all sensors and equipment
# 6. Toggle "Edit Mode" off to switch back to live view
🖱️
Position editor is only needed once per sensor Once you've placed a sensor it stays put in the config file. If you move a physical sensor to a new room, open the editor, select it, and click the new location. Nothing else needs to change — the MQTT topic stays the same.
